# Ролевая модель и схема авторизации

Обновлено: **2026-05-02**.

## Роли

- **Admin (`admin`)** — полный доступ к админке и техническим настройкам.
- **Editor (`editor`)** — управление квестами и этапами без доступа к системным разделам.
- **Участник** — прохождение квеста по ссылке `/play/<access_code>`.

## Актуальная схема входа

Вход для служебных ролей выполняется через **единый экран**: `/admin/login`.

1. Пользователь вводит логин и пароль.
2. По логину определяется роль (`admin` или `editor`).
3. Пароль проверяется по `*_PASSWORD_HASH` (приоритет) или `*_PASSWORD`.
4. Создаётся серверная сессия на 8 часов и cookie `canno_session` (`HttpOnly`, `SameSite=Lax`).
5. После входа: admin перенаправляется на `/admin`, editor — на `/admin/quest/edit`.

## Матрица доступа

| Раздел/действие | Admin | Editor | Участник |
|---|:---:|:---:|:---:|
| Вход через `/admin/login` | ✅ | ✅ | ❌ |
| Главная админка `/admin` | ✅ | ❌ | ❌ |
| Редактор квестов `/admin/quest/*` | ✅ | ✅ | ❌ |
| Импорт/экспорт квестов | ✅ | ❌ | ❌ |
| Аудит/метрики/настройки | ✅ | ❌ | ❌ |
| Прохождение `/play/<access_code>` | ❌ | ❌ | ✅ |

## Переменные окружения

- `CANNO_ADMIN_USER`, `CANNO_ADMIN_PASSWORD`, `CANNO_ADMIN_PASSWORD_HASH`
- `CANNO_EDITOR_USER`, `CANNO_EDITOR_PASSWORD`, `CANNO_EDITOR_PASSWORD_HASH`

Для production используйте только hash-пароли и не храните секреты в git.
