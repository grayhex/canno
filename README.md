# Canno Quest

MVP для офлайн-квестов с уникальной ссылкой участника, этапами, паролями и таймерами.

## Запуск

```bash
export CANNO_ADMIN_USER=admin
export CANNO_ADMIN_PASSWORD='change-me'
python3 app.py
```

Откроется на `http://localhost:8000`.

## P0 статус (критично перед реальными запусками)

### 1) Авторизация и защита админки
- Доступ к `/admin` только после входа через `/admin/login`.
- Есть logout `/admin/logout`.
- Включен rate-limit на попытки логина с одного IP (5 попыток за 5 минут).

### 2) Валидация и безопасность пользовательского ввода
- Ограничены длины полей `username`, `password`, `title`, `prompt` в UI/обработчиках.
- Есть базовая защита brute-force для ввода пароля этапа (по ключу `ip+token`: 10 попыток за 5 минут).

### 3) Надежность хранения данных
- Добавлен migration runner (`schema_migrations`, миграция v1 применяется при старте).
- Добавлен backup-скрипт `backup_db.py`.
- Восстановление: остановить приложение и заменить `canno.db` на backup-файл.

### 4) Ошибки и журналирование
- Используется централизованный `logging` (INFO/WARN/ERROR).
- Добавлены кастомные безопасные страницы для 404/500 и централизованная обработка неожиданных ошибок.

## Переменные окружения

- `CANNO_ADMIN_USER` — логин администратора.
- `CANNO_ADMIN_PASSWORD` — пароль администратора (для dev).
- `CANNO_ADMIN_PASSWORD_HASH` — хэш пароля в формате `pbkdf2_sha256$ITERATIONS$SALT$HASH` (для prod).

## Резервное копирование БД

```bash
python3 backup_db.py --db canno.db --out-dir backups
```

Команда выведет путь созданной копии.
